# 行程卡逻辑分析（神谕.bas ~3690行已注释代码）

> 分析日期：2026-07-04 | 来源：`IQQQ跨码_D据擎2神谕.bas`

## 一、行程卡整体结构

行程卡是VBA中的**综合文本输出引擎**，将各种指标拼接为可读的持仓描述文本。原代码全部被注释，约3690行。

```
程乾坤 = 程隔 + 程周层护型 + 程日层护型 + 程宏周 + 程宏月
```

## 二、分类分析

### A. 周层段芽/根（~80行，lines 3800-3875）
**功能：** 根据柱排+周层段判断操作级别
- **段芽**：ZA>0（DJA之上）→ 日级别操作提示
- **段根**：ZA≤0（DJA之下）→ 周级别操作提示
- 输出：柱排状态 + 冲高提示 + 操作建议

**🔍 参考价值：中**
- 段芽/段根分类与现有的"持主/持被"逻辑一致
- 但柱排的详细程度超过当前策传所需

### B. 周层护型（~60行，lines 3877-3938）
**功能：** 通过周层护型确定操作区域和主升阶段
- 护型首字分类：×=空头, 魅=可试仓, 魑=减仓, 其他=正常
- 包含BTAB正负判断（AB交叉方向）
- 包含ZA>0判断（DJA之上/之下）

**🔍 参考价值：高**
- 护型分类逻辑（×/魅/魑/其他）可作为四域判定的补充
- BTAB+ZA的组合判断可用于<周浮仓>的介入条件

### C. 宏观月类（~15行，lines 3950-3965）
**功能：** 月线级别状态
- 月道尊BTIB（月线IB交叉）
- 月道竖态

**🔍 参考价值：低**
- 月线级别对日级别操作过于粗糙
- 当前策传不需要月线信息

### D. 宏观周类 — 角度（~15行，lines 3969-3981）
**功能：** 周级别角度/护线状态
- 周道护DE部之护（DE护/DE缓）
- 周道护CD部之护
- BTCW（周CW交叉）

**🔍 参考价值：低**
- 角度信息过于细节，对短线操作参考意义不大
- 但DE护/DE缓可作为趋势强度的辅助判断

### E. 宏观周类 — 周奏（~50行，lines 3982-4030）
**功能：** 周级别"奏"状态——主升浪内部结构
- 启DE/启CE（DE/CE交叉启动）
- 首丘/次丘/再丘（主升浪阶段划分）
- 叉DC/叉CB/叉EZ/叉DZ（均线交叉状态）

**🔍 参考价值：高**
- 首丘/次丘/再丘是**主升浪内部阶段划分**，现有策传没有这个维度
- 可用于判断主升浪是否进入尾声（再丘→顶尾）

### F. 宏观周类 — 月阶/战区（~60行，lines 4031-4108）
**功能：** 月阶判定 + 战区划分
- 月阶：0=底震荡, 3/7=主升浪, 4/6=降趋势
- 战区：第0~7战区（基于月阶+BTZD/BTDE）
- 仓限规则：月阶决定最大仓位

**🔍 参考价值：高**
- **月阶**是现有策传没有的宏观周期维度
- **战区**提供了直观的风险定位（"第7战区"比"空长"更有警示效果）
- **仓限规则**（原则1-5）是仓位管理的底层逻辑

### G. 组合输出（~20行，lines 4109-4127）
**功能：** 将上述所有信息拼接为程乾坤
- 程乾坤 = 代码 + 程隔 + 程周层护型 + 程日层护型 + 程宏周 + 程宏月

**🔍 参考价值：低**
- 拼接逻辑简单，不值得单独参考

## 三、值得纳入策传的三项

| 优先级 | 行程卡概念 | 当前策传状态 | 建议 |
|:------:|:----------|:------------|:-----|
| **①** | **月阶+战区** | 无 | 最有价值：月阶=7主升浪 vs 月阶=0底震荡，决定整体策略走向 |
| **②** | **首丘/次丘/再丘** | 无 | 主升浪内部阶段划分，判断是否接近顶尾 |
| **③** | **仓限规则** | 只有静态金额 | 动态仓位上限（月阶→仓限）是仓位管理的核心逻辑 |

| 组合拼接逻辑 | 太简单，不值得单独参考 |
