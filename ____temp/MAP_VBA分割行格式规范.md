# VBA 分割行格式规范

## 概述

本规范定义 VBA 代码中分割行（separator lines）的统一格式，适用于 `昭明计划VS优化.xlsm` 所有模块。

仅约束分割行格式，**不修改任何代码逻辑和注释内容**。

## 分割字符语义

| 字符 | 用途 | 范围 |
|------|------|------|
| `#` | 含标题的大节头部 | 所有模块 |
| `=` | 无标题的分节线 | 所有模块 |
| `-` | 子分组线 | 所有模块 |
| `@` | 大块包装线 | 仅 `OS核管_BS2生成神谕.bas` |

## 核心规则

### 唯一约束：尾部对齐

所有分割行总长统一为 **88 字符**，尾部对齐。

基准线：`'########################################################################################`（0缩进，88字符）

### 缩进

**分割行的缩进保持与所在代码块一致，不做修改。**

即：模块级分割行 0 缩进，函数内分割行跟随其所在代码块的嵌套层级（12/16/20/24/28/32 空格等）。

### 计算公式

> 分割字符数 = 88 - 缩进空格数 - 1（`'` 占位）

| 缩进 | `'` | 分割字符数 | 总长 |
|------|-----|-----------|------|
| 0 | `'` | 87 | 88 |
| 12 | `'` | 75 | 88 |
| 16 | `'` | 71 | 88 |
| 20 | `'` | 67 | 88 |
| 24 | `'` | 63 | 88 |
| 28 | `'` | 59 | 88 |
| 32 | `'` | 55 | 88 |

### 标题行（`#` 专用）

`#` 标题行格式为左右 `#` 夹文字，如：

```
'##################################     工具衍生     ####################################
```

修改时保持缩进不变，文字内容不变，只调整右侧 `#` 数量使总长达 88。

## 修改实例

### 实例1：常量区 `-` 子分组（模块级，0缩进）

```vb
' 修改前（52 / 61 / 11 字符混杂）
'----------------------------------------------------
'指标群：族周段龄
'-------------------------------------------------------------
'指标群：族周龟
'-----------

' 修改后（统一 88 字符，0缩进）
'------------------------------------------------------------------------------------------
'指标群：族周段龄
'------------------------------------------------------------------------------------------
'指标群：族周龟
'------------------------------------------------------------------------------------------
```

### 实例2：函数内 `=` 分割（缩进不变，只调尾部）

当前 76 字符，尾部不齐。修改：缩进保持 12，`=` 从 76 → 75，总长 88。

```vb
' 修改前（12 + 1 + 76 = 89）
            '============================================================================
            '清空原有数据，将每个赋值语句设置空值集中于此
            '============================================================================

' 修改后（12 + 1 + 75 = 88）
            '===========================================================================
            '清空原有数据，将每个赋值语句设置空值集中于此
            '===========================================================================
```

### 实例3：函数内 `-` 子分割（缩进不变，只调尾部）

当前 68 字符。修改：缩进保持 20，`-` 从 68 → 67，总长 88。

```vb
' 修改前（20 + 1 + 68 = 89）
                    '--------------------------------------------------------------------
                    '统计：日段仨属性
                    '--------------------------------------------------------------------

' 修改后（20 + 1 + 67 = 88）
                    '-------------------------------------------------------------------
                    '统计：日段仨属性
                    '-------------------------------------------------------------------
```

### 实例4：函数内深层 `-` 分割（缩进不变，只调尾部）

当前 64 字符。修改：缩进保持 32，`-` 从 64 → 55，总长 88。

```vb
' 修改前（32 + 1 + 64 = 97）
                                '----------------------------------------------------------------
                                '顶型限制
                                '----------------------------------------------------------------

' 修改后（32 + 1 + 55 = 88）
                                '-------------------------------------------------------
                                '顶型限制
                                '-------------------------------------------------------
```

### 实例5：函数内 `@@@@` + `#` 包装块（缩进不变，只调尾部）

```vb
' 修改前（@@@@ 0缩进88字符已对齐，内部 = 12缩进76字符不对齐）
'@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@
            '============================================================================
            ...代码...
            '============================================================================
'@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@

' 修改后（@@@@ 不变，内部 = 缩进12保持，字符数 76→75）
'@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@
            '===========================================================================
            ...代码...
            '===========================================================================
'@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@
```

## 用户代码风格参考（不在此次修改范围内）

以下是对现有代码风格的观察，供理解代码上下文参考，**不属于本次格式修改范围**：

- **中文标识符**：常量、变量、函数名使用中文，具有自文档化特点
- **`醍醐灌顶` 注释**：带时间戳的决策记录，标记重要的设计判断
- **`注` 注释**：带时间戳的技术备注
- **`待完成` / `待研究` 注释**：标记未完成的工作项
- **注释掉的代码**：保留的探索历史，不删除
- **大函数**：有意的线性流程，不拆分
- **分隔行层次**：常量区 `-` 的不同长度（11/52/61）暗示指标群 > 子组 > 条目的三级视觉层次
