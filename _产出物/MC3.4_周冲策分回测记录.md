## 2026-07-09 第三轮: 真代双高验证 — VBA公式直接驱动

**方法:** VBA源码 `位os临幅HR = (临高/结收-1)*100` 定义了HR本周临(col47)相对于本周首日结收的百分比。PR本周临(col46)同理。

```vb
' VBA原始公式:
ARROS(位os临幅PR) = Round((ARROS(位os临收) / ARROS(位os结收) - 1) * 100, 2)
ARROS(位os临幅HR) = Round((ARROS(位os临高) / ARROS(位os结收) - 1) * 100, 2)
```

**公式推导:**
- 结收 = 本周首日收盘价(上周五收盘)
- 临收 = 本周每日收盘价(累积)
- 临高 = 本周每日最高价(累积)
- PR本周临 = (临收/结收-1)*100 = 本周累积收盘收益率
- HR本周临 = (临高/结收-1)*100 = 本周累积最高收益率(已是running max)

**对于下周冲高:**
- 下周结收 = 本周收盘
- 下周HR = (下周最高价/下周结收-1)*100 = (下周最高价/本周收盘-1)*100 = **真高幅** ✓
- 真高幅 = 下周HR本周临(最后一天) — 直接使用col47值
- 代高幅 = max(下周PR本周临) — 取col46的周内最大值(PR不是running max)

**全量数据:** 48323周, 89文件(10指数+4宽基+45行业ETF+30个股)

### 基准(全部周 48323周)

| 指标 | 成功率 | 均幅 | ≥3% | ≥5% |
|:----|:-----:|:----:|:---:|:---:|
| **真高幅**(盘中最高) | **91%** | **+3.74%** | **42%** | **25%** |
| 代高幅(最高收盘) | 73% | +2.32% | 31% | 17% |

### 条件对比 (按真冲高排序)

| 条件 | 样本 | 真冲高 | 真均幅 | 真≥3% | 代冲高 |
|:-----|:---:|:------:|:------:|:-----:|:------:|
| 全部周(基准) | 48323 | 91% | +3.74% | 42% | 73% |
| **ZA 7~10+升排+非孕** | **823** | **94%** | **+4.94%** | **52%** | 79% |

**关键解读:**
- 真冲高基准91%: 5天中至少有一天盘中翻红的概率

## 历史结果差异说明（2026-07-09 排查结论）

**现象：** 旧回测(06-30~07-02)的最优条件成功率91%，新回测(07-09)同样条件仅52%。

**排查过程：**
1. ✅ 列索引修复：旧脚本误用col5(主七=1-5)当涨跌幅，新脚本用col9(日涨)
2. ✅ 两种方法对比：`结价比(col36首日→下周一col36首日)` vs `日涨累积(col9)` 均差异仅0.05pp
3. ✅ 确认新脚本的周收益计算正确。

**根本原因 — 旧脚本逻辑错误：**

旧脚本 `验证_60标的.py` 中：
```python
rets.append((nw['close']-w['close'])/w['close']*100)
```
其中 `w.close = col36_week[i]_first * F`，`nw.close = col36_week[i+1]_first * F`

col36_week[i]_first = 结价周前(i周首日) = i-1周周五收盘
col36_week[i+1]_first = 结价周前(i+1周首日) = i周周五收盘

**所以 ret = (i周收盘 - i-1周收盘) / i-1周收盘 = i周自身的收益，不是i+1周的收益！**

旧脚本实际测的是 **条件满足当周自身的涨跌**（同周期相关性=91%）：
- 条件(金升+甲乙+ZA<5+升排+非孕)本身就是本周的强劲趋势信号
- 所以条件成立时本周自身上涨概率高（91%）

新脚本测的是 **条件满足后下一周的涨跌**（跨周期预测力=52%）：
- 周级别任何条件对下周预测成功率都在50~55%之间
- 结论：**周级别信号对预测下周基本无效**

**对策略的影响：**
- 旧结论（91%成功率）→ 应纠正为：条件与本周走势高度相关，**但不预示下周**
- 周级别信号只能用于 **确认当前趋势**（同周期判断），不能用于 **预测下周方向**
- 周冲周冲策分卡的核心价值转向 **区分期望收益大小**而非追求高胜率
- 下日冲高(日级别信号)是否有效需另验证（不在本次范围）
