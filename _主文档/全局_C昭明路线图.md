### 1.4 系统化措施如何避免人性陷阱
> A盯盘警示卡列出了常见心理陷阱（如"再扛一下""先买一点试试"等），
> 本路线图的系统化手段正是为了从制度上杜绝这些陷阱——**不止是提醒，而是让陷阱根本不存在操作空间。**
> **注：** 下表为系统化阻断措施映射（C文件职责），非心理陷阱内容重复（A文件职责）。<br>
> 目的：展示每条心理陷阱对应的系统化阻断方案，证明"制度上杜绝"的可行性。

| 人性陷阱 | A警示卡描述 | C系统化措施 | 阻断机制 |
|---------|-----------|------------|---------|
| "再扛一下，马上反弹" | DJE破了就是破了 | §四(1) DJE跌破强制清仓 → MC4自动化 | 代码自动执行，人不可撤销 |
| "先买一点试试" | 试=亏损的开始 | §四 MC3 系统综合评分＞80才推荐 | 不推荐就不买，无手动买入入口 |
| "这个票感觉要涨" | 感觉没用，看WXCD | §四 MC3 系统评分+策传输出才入候选 | 不看感觉，看量化评分 |
| "追高搏一把" | DSHA高时只持基仓 | §五(3) DSHA追高限制 → MC4浮仓判断自动执行 | 系统限制浮仓开仓，非人为判断 |
| "被震出去" | 尊从算法，不要被震出 | §八 MC4 持仓决策树固定输出 | 决策树决定持有/清仓，不因盘中波动改变 |
| "柱子排列看着要涨" | 不要按柱排期望 | §四 MC1-MC3 回测验证后才用于选股 | 历史数据验证了的信号才纳入决策，非主观判断 |

> **核心思路：** A把人性弱点写在屏幕上让你警醒；C把这些弱点对应的**系统化阻断措施**逐一落地。<br>

### 2.2 DXZE>0 区域的完整划分

需要说明DJE的重要性。DJE-WJC-MJA，是所有做多策略的逻辑底线。

需要对 DXZE>0 区域通过纯理论分析或历史遍历，分类并归纳各种走势的后续概率。

#### 2.2.1 公理

**公理一：** 所有浮仓的操作区域（入仓、持仓），都在 DJEDC 之上。

> **注一：** 下破 DJC 应将浮仓退出；当处于 DJC 之下且 DJE 之上，只可能是 `_多长` 基仓。DXCD<0 属于周级别 WXAB<0，当未上破 DJD 时不应介入，应选择即将导致 DXCD 正交的情况。

#### 2.2.2 重点分类依据与初步结论

**重点分类依据：**

- DXCD（日级核心框架方向）
- DXBC（日级BC交叉状态）
- DXAB（日级护型正交状态）
- DJA之上柱排（升/跌/人）
- DJA之上中符（顶/哼/哈/底）
- 日级别波型

**初步结论：**

| 条件 | 适用浮仓类型 | 说明 |
|:-----|:-----------|:-----|
| DXZA≥0 | **日级别浮仓**介入点 | 站上DJA，可用<日冲策略> |
| DXZC≥0 | **周级别浮仓**介入点 | 站上DJC，可用<周冲策略> |
| 首下破DJA（DTZA=-1） | **周级别浮仓（特例）** | 不应视为日线机会，应归为周级别决策 |
| DXZC<0 | **禁止介入浮仓** | 等待DXZC>0后再介入。不是"下破产生的机会"，而是"下破后止跌走稳，且存在周柱排列等条件才制造上涨机会" |

> 注意：不要认为下破DXZC是机会。真正的机会来自**下破后止跌→走稳→回弹→回踩确认**这一系列信号，而不是下破本身。

#### 2.2.7 日柱排列的介入时机

### 2.3 专题一：三策略体系（月基/周冲/日冲）

#### 2.3.1 月基策略 — 长线底仓

| 维度 | 内容 |
|:----|:------|
| **核心问题** | 这股票适合长期持有吗？ |
| **评估工具** | 月基评分（年化×胜率×盈亏比×均单次/震仓） |
| **入场** | 四域=多长时介入 |
| **出场** | WXZC<0（趋势终结），不止盈 |
| **持仓周期** | 月级别，平均7.1周 |
| **仓位性质** | 底仓（长线持仓），占资金池主要部分 |

**月基策略的核心逻辑：**
```
多长开始 → 买入 → 持有（期间无视周/日波动）→ WXZC<0 → 卖出
   ↑                        ↑                         ↑
   入场条件                  93.9%续涨率                趋势终结
   策分>=85                稳定无衰减
```

**操作规则：**
```
- 买入后持有不动，不止盈、不做T
- 每周检查WXZC状态
- WXZC<0时无条件卖出
- 月基评分≥0.8（仁慈级）优先建仓
```

#### 2.3.2 周冲策略 — 周线波段
| 维度 | 内容 |
|:----|:------|
| **核心问题** | 下周冲高概率多大？ |
| **评估工具** | 周冲评分（四域20+ZA30+柱排25+波型25） |
| **入场** | 上周五尾盘或本周初（周一）买入 |
| **出场** | 本周内冲高卖出，最迟周五收盘前 |
| **持仓周期** | 1周（周初→周内冲高时） |
| **仓位性质** | 波段仓位，占用资金池剩余部分 |
**周冲策略的核心逻辑：**

```
周初 → 检查周冲评分 → 评分>=70？ → 买入 → 等待本周冲高 → 卖出
   ↑                     ↑                    ↑            ↑
   周一评估             策分85+最优            93.9%概率    冲高出
                        策分70-84次优          冲高达标    或周五出
```

**操作规则：**
```
- 每周一检查候选标的的周冲评分
- 评分≥85（A级）优先操作，评分70~84（B级）辅助操作
- 买入后本周内任一天冲高达标即卖出
- 本周五收盘前无论是否达标必须卖出
- 同一标的可与月基策略共存（月基底仓不动，周冲仓做波段）
```

#### 2.3.5 三策略关系与选择优先级

**关系图：**
```
四域=多长
    │
    ├── 月基策略（底仓）
    │     └── 买入后持有，直到WXZC<0
    │
    └── 周级别信号
          │
          ├── 周冲策略（波段）
          │     └── 上周五或本周初买入，冲高卖出
          │
          └── 日级别信号
                └── 日冲策略（脉冲）
                      └── 前日尾盘买，次日冲高出
```

**核心原则：**
1. **优先级：月基 > 周冲 > 日冲**（月基确定性最高，日冲最低）
2. **资金池共享：** 月基用完了额度，周冲/日冲不应再开
3. **可共存：** 同一标的可同时持有月基底仓 + 操作周冲仓位（互不冲突）
4. **月基不止盈：** 月基的收益来自长期趋势，不是短线冲高
5. **周冲/日冲必须止盈：** 到时不卖就变成长线套牢，违背策略初衷

## 四、仓位管理：理论与实施

> 仓位管理涉及持仓组合、板块轮动和ETF配置，三者共同决定资金在各类资产上的分配。

### 4.1 持仓组合设计

#### 4.1.4 仓位分配

**仓位分配：**

| 策略 | 资金占比 | 标的数量 |
|:----|:--------:|:--------:|
| 月基策略（底仓） | 60~80% | 3~5只 |
| 周冲策略（波段） | 15~25% | 2~4只 |
| 日冲策略（脉冲） | 5~10% | 1~2只 |
| 现金 | 5~10% | — |

#### 4.1.1 问题：单一标的的持仓面临系统性风险，若集中在一个板块，一旦该板块轮动结束，整体回撤难以控制。

> 解决方案：持仓组合设计，保证在不同板块持有不同标的，仓位受限制，确保市场无论如何波动，总有某些板块可以上涨。

#### 4.1.2 核心原则：

- 单板块仓位上限 ≤ 30%（避免板块集中风险）
- 单标的仓位上限 ≤ 15%（避免个股风险）
- 必须持有至少 **3个不同板块** 的标的
- 持有一定比例的 **宽基ETF**（如沪深300ETF、中证500ETF），跟随板块轮动

**组合结构（总仓位上限=100%，基仓与浮仓共享同一资金池）：**

| 仓位类型 | 标的数量 | 仓位比例 | 说明 |
|:--------|:--------|:--------|:-----|
| **\_多月基仓位**（含ETF） | 3~5只个股 + 1~2只ETF | 60~80% | 基仓：分散在不同板块，\_多长个股+宽基ETF |
| **浮仓**（<周冲仓位>+<日冲仓位>） | 2~4只 | 15~25% | 浮仓：在基仓保护下做冲高脉冲，占用基仓余额 |
| **现金** | — | 5~10% | 备用，等待机会 |

> **资金池说明：** 基仓和浮仓共享同一资金池，非叠加关系。总仓位 = 基仓仓位 + 浮仓仓位 ≤ 100%。<br>
> 基仓满仓（80%）时，浮仓最多占 20%；基仓轻仓（60%）时，浮仓可占 25%。<br>
> 浮仓下单时从基仓余额中支取，平仓后资金归还基仓余额。

#### 4.1.3 挿仓组合设计（板块轮动跟随机制）：

1. 宽基ETF自动跟随大盘，保证不踏空
2. 个股分散在不同板块，某板块走弱时其他板块可能走强
3. 当系统判定某板块进入\_多长区域时，可适度增加该板块配置
4. 当系统判定某板块进入\_空长区域时，减仓该板块，转配ETF

**持仓调整触发条件：**

- 区划变化（\_多长→\_多被→\_空看→\_空长）
- 板块轮动信号（板块指数进入\_多区域）
- 单板块仓位超限
