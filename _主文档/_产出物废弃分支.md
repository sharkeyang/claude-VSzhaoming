### 1.1 原目标

将 VBA 的历史回测生成引擎（1090列日线数据）迁移至纯 Python，摆脱对 Excel COM 的依赖，实现批量快速回测。

### 1.3 COM 调用替代方案（当前方案）

```
Python → excel.Application.Run("XL算展取谕组", 代码) → VBA计算 → 输出CSV → Python读取
```

| 原VBA模块 | 调用方式 | 说明 |
|:----------|:---------|:-----|
| 核1(EMA/BTZ) | `XL算展数程跨期_生成历史` | VBA保留，Python通过COM调用 |
| 核2(策略计算)+核3(柱排) | `IQQQ跨码_D据擎2神谕` | VBA保留，Python通过COM调用 |
| 核5+6(生成算展) | `XL算展格程跨期` | VBA保留，Python通过COM调用 |

**优势：** 一致率100%，Python只需一行调用，无需维护双引擎。
**代价：** 依赖 Excel COM（需安装 Excel + 启用 VBA 访问），单标的约10秒。

### 1.4 经验教训

1. **VBA 位谕系统过于复杂**：1092列中日级别约500+列，大量派生列（如 WXCD 依赖前N周的 BTZ/JZ 值链），Python 复现成本极高
2. **益盟公式不可替代**：WXCD/护型等核心列由益盟公式计算，益盟的公式引擎与 VBA/VBA 内嵌实现存在差异
3. **COM 调用是正确路径**：在无法完全脱离 Excel 的前提下，直接调用 VBA 是最可靠的方式

## 2. 从CSV结算的尝试与结论

**状态：** ❌ 所有路径均失败（2026-06-13 ~ 2026-06-18）
**目标：** 加速 VBA 全量结算中的 CSV 解析环节（全量结算基线 ~10-15 分钟）

### 尝试过的路径

| 方向 | 结果 | 日期 |
|:----|:----|:----:|
| CSV 3遍读取 → 1遍合并读取 | ❌ 慢 40~84% | 2026-06-13 |
| Python 预加载 CSV 到内存 | ❌ 持平（±2%） | 2026-06-13 |
| VBA 内存字典缓存 | ❌ 持平或更慢 | 2026-06-13 |
| ReDim Preserve 动态数组 | ❌ 更慢 | 2026-06-13 |
| 优化 CSV 解析函数 | ❌ 慢 25~29% | 2026-06-18 |

### 根因

VBA 的 CSV 解析是 **CPU 密集型，不是 I/O 密集型**。`Line Input + Split + CDate/CDbl` 的解析开销远大于文件读取的 I/O 开销。任何增加每行解析工作量的"优化"反而更慢。

### 当前不可优化的四大约束

1. **除权除息** → AKshare 必须全量下载（不可增量）
2. **1090 列全量计算** → 用户保留全部列（不可减列）
3. **Python 重写 VBA 逻辑** → 一致率风险（精确度优先）
4. **VBA CSV 解析** → CPU 已到上限（全部优化已试）

**结论：** 10-15 分钟是 VBA 全量结算的基线成本，已明确接受，无进一步优化空间。

## 3. 花册进行数据库改造的尝试与结论

**状态：** ⏹️ 已废止（伪优化）
**目标：** 将花册.xlsx（持仓/候选池数据）迁移至 SQLite/数据库

### 分析结论

**花册转数据库是伪优化** — 它连前5名瓶颈都进不去。花册.xlsx 读取仅占全链路的 **<0.1% 耗时**，转数据库即使提升100倍，对总耗时的影响也不到0.1分钟。

### 全链路耗时排名

| 排名 | 环节 | 耗时 | 占比 | 可优化？ |
|:----:|:-----|:----:|:----:|:--------:|
| 1 | AKshare 全量下载 | ~56 分钟 | 76% | ❌ 除权除息必须全量 |
| 2 | VBA 全量结算(含神谕) | ~10-15 分钟 | 18% | ❌ 全部优化路径已试 |
| 3 | 策传看板刷新周期 | ~75 秒 | 2% | ⚠️ 可调但影响不大 |
| 4 | 策传生成器 COM 读取 | ~60-120 秒 | 2% | — |
| 5 | Python 选股引擎 | <30 秒 | <1% | — |
| 6 | **花册.xlsx 读取** | **1~3 秒** | **<0.1%** | **❌ 没必要优化** |

**结论：** 花册转数据库 → **不执行**。真正的瓶颈是 AKshare 下载（76%）和 VBA 结算（18%），而这两项均已证实无可优化空间。
